1 About



I love spacemacs a lot because it combines key-bindings/modal concepts of Vim (my personal favorite reason) and functionality of Emacs via the beautiful lisp (emacs-lisp) language.

Kinda best of both the worlds! Ehh, right?

In my free-time, I try to learn more about the language and the editor itself bit-by-bit. Learning about the functionalities and packages, thus configuring it to my own personal taste is the favorite part.

Here is the experience so far…

2 Basic setup

To try-out my spacemacs configuration, create a directory at home as .spacemacs.d and move the .spacemacs file from home to the newly created directory and rename it as init.el. 

mkdir ~/.spacemacs.d
mv ~/.spacemacs ~/.spacemacs.d/init.el

2.1 Dependencies

For a smooth experience, you need to have the following bare-minimum software installed on your system and ensure they are updated.

2.1.1 Emacs

Minimum version required: 25.0.1. Download it via: 

sudo apt install emacs

# Explicitly download emacs with version 25.0.1.
sudo apt install emacs25

2.1.2 Spacemacs

You also need to clone the spacemacs github repository in your home directory (recommended).

Stable version 

git clone https://github.com/syl20bnr/spacemacs ~/.emacs.d

Develop version

For more details, please refer the spacemacs installation guide. 

git clone -b develop https://github.com/syl20bnr/spacemacs ~/.emacs.d

I use Develop version of the spacemacs.

Now open the init.el file and under the dotspacemacs/user-config section, we need to add few changes as follows: 

(defun dotspacemacs/user-config ()
 "Configuration for user code:
This function is called at the very end of Spacemacs startup, after layer
configuration.
Put your configuration code here, except for variables that should be set
before packages are loaded."

   ;; Load the directory for custom spacemacs config files.
   (push "~/.spacemacs.d/" load-path)

   (load "<config-file-name#1>")
   (load "<config-file-name#2")
   ;; and so on...

 ;; rest of the code
 )

We are almost there. The only thing we need is to download the following files under ~/.spacemacs.d directory and edit minor changes in init.el file as suggested and we are good to go!

3 C/C++ configurations

3.1 About

Enable flycheck integration along-with cmake-ide in c-c++ mode.

Also integrate clang-analyzer (a static code analysis tool) along-with cppcheck for strict syntax-checking. (read flycheck config).

Removed clang-format-before-save hook from c-c++ mode as it was causing auto-format issue in other modes. So to use it, one has to manually call the function via SPC m = =.

3.2 Setup

To use this configuration, you would need to download the following *.el file in the correct path as well as update the init.el file as describe below:

Location ~/.spacemacs.d/c-c++config.el
Link c-c++config.el

Add (load "c-c++config") to the dotspacemacs/user-config section of your dotfile.

Example: 

(defun dotspacemacs/user-config ()
 "Configuration for user code:
This function is called at the very end of Spacemacs startup, after layer
configuration.
Put your configuration code here, except for variables that should be set
before packages are loaded."

   ;; Load the directory for custom spacemacs config files.
   (push "~/.spacemacs.d/" load-path)

   (load "c-c++config")

 ;; rest of the code
 )

And the ~/.spacemacs.d directory should look something like this:

4 Flycheck (syntax-checking) configurations

4.1 About

4.2 Setup

To use this configuration, you would need to download the following *.el file in the correct path as well as update the init.el file as describe below:

Location ~/.spacemacs.d/flycheckconfig.el
Link flycheckconfig.el

Add (load "flycheckconfig") to the dotspacemacs/user-config section of your dotfile.

Even though downloading clang-analyzer is optional, it’s highly recommend along-side other tools to reduce bugs in your code and improve its quality.

Download clang-analyzer~ via: 

sudo apt install clang-analyzer

For fully-featured experience, download the complete clang-package via: 

sudo apt install clang clang-format clang-analyzer
